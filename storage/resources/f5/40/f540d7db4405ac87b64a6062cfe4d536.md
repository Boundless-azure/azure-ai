## 基础约定

| 项 | 值 |
|---|---|
| Base URL | `https://game-chedun.doutrend.cn/api/terminal` (生产) · 本地开发可走 `http://localhost:<APP_PORT>/api/terminal` |
| 字符集 | UTF-8 |
| 请求格式 | `Content-Type: application/json` (POST) |
| 响应格式 | `Content-Type: application/json` |
| HTTP 状态码 | **业务接口固定 200**;5xx 仅当服务端基础设施故障 |
| 时区 | 时间字段一律 ISO 8601 (`2026-05-17T08:30:00.000Z`),后端 UTC |
| ID | `userId` 为整数 (≥1,自增从 1000 起,默认测试用户 = 10000);`runId` 为 UUID;`deviceId` 由终端定义 (字符串 ≤ 128) |

---

## 统一响应包络

成功:

```json
{ "code": 0, "message": "OK", "data": { ... } }
```

业务失败 (HTTP 仍 200):

```json
{ "code": 40002, "message": "游戏币不足", "data": null }
```

服务端故障 (HTTP 500):

```json
{ "code": -1, "message": "内部错误", "data": null }
```

终端逻辑伪代码:

```js
if (httpStatus !== 200) panic('服务器异常');
if (response.code !== 0) handleBusinessError(response.code, response.message);
const data = response.data;
```

## 鉴权

所有终端接口由 `TerminalAuthGuard` 守卫,二选一即可:

| 位置 | 字段 | 示例 |
|---|---|---|
| Header (**推荐**) | `X-Sign-Secret` | `X-Sign-Secret: F4VG5YG2…oZ-…` |
| Query | `secret` | `?secret=F4VG5YG2…oZ-…` |

值取自后端 `.env` 的 `GAME_SIGN_SECRET`。错误或缺失 → `10002 SIGN_SECRET_INVALID`。

> 不读 body 是为了让 DTO 严格白名单不被破坏。

---

## 错误码总表

| code | 常量 | 含义 | 出现场景 |
|---|---|---|---|
| `0` | SUCCESS | 成功 | — |
| `10001` | PARAM_INVALID | 参数错误 | DTO 校验失败 / 未知路由 |
| `10002` | SIGN_SECRET_INVALID | 密钥错误 | header / query 缺失或不匹配 |
| `20001` | USER_NOT_FOUND | 用户不存在 | `device/bind` 时 userId 未注册 |
| `30001` | DEVICE_NOT_BOUND | 设备未绑定 | 任何 deviceId → user 反查路径 |
| `40001` | GAME_NOT_FOUND | 游戏不存在 | gameCode 在 games 表中找不到 |
| `40002` | INSUFFICIENT_COINS | 游戏币不足 | 起局 / 负向调整 余额会变负 |
| `40003` | ACTIVE_RUN_EXISTS | 已有进行中游戏 | 起局时旧 run 还在时限内 |
| `40004` | CONSECUTIVE_LIMIT_REACHED | 达到连续游玩上限 | 起局时连续段长度 ≥ 限制 |
| `40005` | RUN_NOT_FOUND | run 不存在 | 结束 / 状态查询 |
| `40007` | COIN_TIP_REQUIRED | 必须填写说明 | `coin/adjust` 的 tip 为空 |
| `-1` | SERVER_ERROR | 服务端错误 | HTTP 500,基础设施故障 |

## HTTP 端点

### POST /api/terminal/device/bind

**绑定设备到用户**。`deviceId` 已绑则覆盖原 userId(满足"下一个绑定覆盖上一个")。

**请求**

```http
POST /api/terminal/device/bind
X-Sign-Secret: <GAME_SIGN_SECRET>
Content-Type: application/json

{
  "userId":  "10000",
  "deviceId": "73F156FD"
}
```

**Payload 字段**

| 字段 | 类型 | 必填 | 校验 | 说明 |
|---|---|---|---|---|
| `userId` | int | ✅ | ≥ 1 | 用户主键 (整数, 默认测试用户 = 10000) |
| `deviceId` | string | ✅ | 长度 1–128 | 设备标识 |

**成功响应 data**

```json
{ "deviceId": "73F156FD", "userId": "10000" }
```

**可能错误**

| code | 何时 |
|---|---|
| 10001 | `userId` 非正整数 或 `deviceId` 为空 / 超长 |
| 10002 | secret 错误 |
| 20001 | `userId` 在数据库中不存在 |

---

### POST /api/terminal/device/unbind

**解绑指定设备**。幂等(无记录也返回成功)。

**请求**

```http
POST /api/terminal/device/unbind
X-Sign-Secret: <GAME_SIGN_SECRET>
Content-Type: application/json

{ "deviceId": "73F156FD" }
```

**Payload 字段**

| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| `deviceId` | string | ✅ | 长度 1–128 |

**成功响应 data**

```json
{ "deviceId": "73F156FD" }
```

**可能错误**

| code | 何时 |
|---|---|
| 10001 | `deviceId` 为空 |
| 10002 | secret 错误 |

### GET /api/terminal/game/:gameCode

**查询游戏元数据**(开局币、连续限制等)。

**请求**

```http
GET /api/terminal/game/g1
X-Sign-Secret: <GAME_SIGN_SECRET>
```

**Path 参数**

| 参数 | 类型 | 说明 |
|---|---|---|
| `gameCode` | string | 游戏 code (UNIQUE),由管理员录入 |

**成功响应 data**

```json
{
  "id": "f0a7…uuid",
  "code": "g1",
  "name": "猜数字",
  "description": "经典 1-100 猜数",
  "costCoins": 10,
  "maxConsecutivePlays": 3,
  "createdAt": "2026-05-17T08:00:00.000Z",
  "updatedAt": "2026-05-17T08:00:00.000Z"
}
```

**可能错误**

| code | 何时 |
|---|---|
| 10002 | secret 错误 |
| 40001 | `gameCode` 找不到 |

### POST /api/terminal/run/start

**起局**(单事务内完成 4 件事:超时活跃 run 自动结束 → 校验连续游玩限制 → 扣开局币 → 创建 run + socket 广播 `started`)。

**请求**

```http
POST /api/terminal/run/start
X-Sign-Secret: <GAME_SIGN_SECRET>
Content-Type: application/json

{
  "deviceId": "73F156FD",
  "gameCode": "g1"
}
```

**Payload 字段**

| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| `deviceId` | string | ✅ | 长度 1–128 |
| `gameCode` | string | ✅ | 长度 1–64 |

**成功响应 data**

```json
{
  "runId": "5f8d…uuid",
  "costCoins": 10,
  "balance": 90,
  "startedAt": "2026-05-17T08:30:00.000Z"
}
```

**可能错误**

| code | 何时 |
|---|---|
| 10001 | 参数缺失或超长 |
| 10002 | secret 错误 |
| 30001 | `deviceId` 未绑定 |
| 40001 | `gameCode` 不存在 |
| 40002 | 玩家余额 < `game.costCoins` |
| 40003 | 当前已有活跃 run 且未超时 (`lastActivityAt` 距今 < `RUN_TIMEOUT_MINUTES`) |
| 40004 | 同游戏连续段长度 ≥ `maxConsecutivePlays` 且间隔 < `RUN_CONSECUTIVE_RESET_MINUTES` |

> 当旧 active run 已超过 `RUN_TIMEOUT_MINUTES`(默认 5 分钟)无心跳,起局会**自动**把它标记为 `timeout` 并继续开新局,不会报 40003。

### POST /api/terminal/run/end

**结束当前活跃 run 并结算分数**(通过 `deviceId` 反查 active run)。

**请求**

```http
POST /api/terminal/run/end
X-Sign-Secret: <GAME_SIGN_SECRET>
Content-Type: application/json

{
  "deviceId": "73F156FD",
  "score": 1234
}
```

**Payload 字段**

| 字段 | 类型 | 必填 | 校验 |
|---|---|---|---|
| `deviceId` | string | ✅ | 长度 1–128 |
| `score` | int | ✅ | 任意整数 (允许负) |

**成功响应 data**

```json
{
  "runId": "5f8d…uuid",
  "score": 1234,
  "balance": 90
}
```

> 结束接口本身不调币,`balance` 仅展示当前快照。如要按胜负发奖,请在 `run/end` 前/后调 `coin/adjust`。

**副作用**:服务端向 `run:{runId}` 房间广播 `ended` 事件后 `disconnectSockets`,清场房间。

**可能错误**

| code | 何时 |
|---|---|
| 10001 | `score` 非整数 |
| 10002 | secret 错误 |
| 30001 | `deviceId` 未绑定 |
| 40005 | 该用户当前没有 active run |

### POST /api/terminal/coin/adjust

**增减游戏币**(`tip` 必填,负 delta 不能透支)。在 run 中调用会刷新 `lastActivityAt` 并广播 `coin-change`。

**请求**

```http
POST /api/terminal/coin/adjust
X-Sign-Secret: <GAME_SIGN_SECRET>
Content-Type: application/json

{
  "deviceId": "73F156FD",
  "delta": -5,
  "tip": "失败惩罚",
  "gameCode": "g1"
}
```

**Payload 字段**

| 字段 | 类型 | 必填 | 校验 | 说明 |
|---|---|---|---|---|
| `deviceId` | string | ✅ | 长度 1–128 | 反查 user |
| `delta` | int | ✅ | 非零整数 | 正加负减 |
| `tip` | string | ✅ | 长度 1–255,非空 | 变动原因,审计可见 |
| `gameCode` | string | ❌ | 长度 1–64 | 关联游戏,缺省按 active run 推断 |

**成功响应 data**

```json
{
  "balance": 85,
  "change": {
    "id": "9b1f…uuid",
    "userId": "10000",
    "delta": -5,
    "reason": "terminal-adjust",
    "tip": "失败惩罚",
    "gameId": "f0a7…uuid",
    "runId": "5f8d…uuid",
    "balanceAfter": 85,
    "createdAt": "2026-05-17T08:35:00.000Z"
  }
}
```

**副作用**:在 active run 中时,服务端向 `run:{runId}` 广播 `coin-change`。

**可能错误**

| code | 何时 |
|---|---|
| 10001 | 参数缺失或 `delta` 非整数 |
| 10002 | secret 错误 |
| 30001 | `deviceId` 未绑定 |
| 40001 | 显式传了 `gameCode` 但不存在 |
| 40002 | `delta < 0` 且会导致余额变负 |
| 40007 | `tip` 为空或纯空白 |

---

### GET /api/terminal/run/:runId/status

**轮询 run 状态最小集**(用于 socket 断线后补查)。**仍需带 secret**。

**请求**

```http
GET /api/terminal/run/5f8d…uuid/status
X-Sign-Secret: <GAME_SIGN_SECRET>
```

**Path 参数**

| 参数 | 类型 |
|---|---|
| `runId` | UUID |

**成功响应 data**

```json
{
  "runId": "5f8d…uuid",
  "status": "active",
  "startedAt": "2026-05-17T08:30:00.000Z",
  "endedAt": null,
  "score": null
}
```

`status` 取值:

| 值 | 含义 |
|---|---|
| `active` | 进行中 |
| `finished` | 终端正常 end |
| `timeout` | 未及时 end,被下一次 start 时回收 |
| `aborted` | 管理员强制结束 |

**可能错误**

| code | 何时 |
|---|---|
| 10002 | secret 错误 |
| 40005 | runId 不存在 |

## Socket.io: /run_notify

实时通知服务,namespace `/run_notify`,以 `runId` 为房间维度。

### 连接

```js
const sock = io('https://game-chedun.doutrend.cn/run_notify', { transports: ['websocket'] });
sock.on('connect', () => {
  sock.emit('join', { runId: '<起局返回的 runId>' });
});
```

> 当前 socket 不强制鉴权,**任何能连到 host 的客户端都能听某 run 的事件**。后续会加 token 校验。

### 服务端事件

| 事件 | 何时 | Payload 样例 |
|---|---|---|
| `started` | run 创建完成 | `{ runId, costCoins, balance, startedAt }` |
| `coin-change` | run 期间 `coin/adjust` 触发 | `{ runId, delta, balance, tip }` |
| `ended` | `run/end` 正常结束 (之后服务端会断开本房间) | `{ runId, score, balance }` |
| `aborted` | 管理员强制结束 (之后断开本房间) | `{ runId, reason: 'admin', tip }` |
| `timeout` | 下一次起局触发了对超时 run 的回收 (之后断开本房间) | `{ runId }` |

### 终端推荐处理

- 起局后立即 `emit('join', { runId })`
- 监听 `ended` / `aborted` / `timeout` → 视为终局,清理本地状态
- 监听 `coin-change` → 刷新本地余额展示
- 断线时退化到轮询 `/api/terminal/run/:runId/status`

## 完整调用流

```
┌─────────────┐      bind          ┌────────────────────┐
│ 1. 接入设备 │ ──────────────────▶│ /device/bind       │
└─────────────┘                    └────────────────────┘
┌─────────────┐      get game       ┌────────────────────┐
│ 2. 查游戏元 │ ──────────────────▶│ /game/:gameCode    │
└─────────────┘                    └────────────────────┘
┌─────────────┐      start run     ┌────────────────────┐    socket
│ 3. 开局     │ ──────────────────▶│ /run/start         │ ──────────▶ started
└─────────────┘                    └────────────────────┘
                                                          (期间)
┌─────────────┐      adjust         ┌────────────────────┐    socket
│ 4. 扣/奖币  │ ──────────────────▶│ /coin/adjust       │ ──────────▶ coin-change
└─────────────┘                    └────────────────────┘
┌─────────────┐      end run       ┌────────────────────┐    socket
│ 5. 结束结算 │ ──────────────────▶│ /run/end           │ ──────────▶ ended + 断房间
└─────────────┘                    └────────────────────┘
┌─────────────┐      unbind        ┌────────────────────┐
│ 6. 解绑设备 │ ──────────────────▶│ /device/unbind     │
└─────────────┘                    └────────────────────┘
```

## 测试快速开始

> 生产域名 `https://game-chedun.doutrend.cn`,secret 取生产 `.env` 的 `GAME_SIGN_SECRET`。
> 本地开发把 `HOST` 换成 `http://localhost:3006`。
>
> 数据库已通过 [seed migration](../database/migrations/1779100000000-SeedDefaultData.ts) 写入默认用户 `10000` (初始 9999 币)、默认游戏 `default-game`、默认绑定 `deviceId=73F156FD`。

```bash
SECRET='F4VG5YG2aUgP88LqAzDHh-03cdATsIkK0f6R2oZ-qSmBulc_5w3_Xp_KTuYey-Ow'
HOST='https://game-chedun.doutrend.cn'
BASE="$HOST/api/terminal"

# 默认用户/游戏/绑定迁移已写入, 跳过 setup, 直接试

# 1. 查默认游戏
curl -s -H "X-Sign-Secret: $SECRET" "$BASE/game/default-game"

# 2. (可选) 调整默认用户余额, 比如再加 500
curl -s -X POST "$HOST/api/admin/coin/adjust" \
  -H "X-Sign-Secret: $SECRET" -H "Content-Type: application/json" \
  -d '{"userId":10000,"delta":500,"tip":"测试加币"}'

# 3. 起局
curl -s -X POST "$BASE/run/start" \
  -H "X-Sign-Secret: $SECRET" -H "Content-Type: application/json" \
  -d '{"deviceId":"73F156FD","gameCode":"default-game"}'
# 拿到 data.runId 记下来

# 4. 期间扣币
curl -s -X POST "$BASE/coin/adjust" \
  -H "X-Sign-Secret: $SECRET" -H "Content-Type: application/json" \
  -d '{"deviceId":"73F156FD","delta":-5,"tip":"失败惩罚","gameCode":"default-game"}'

# 5. 结束
curl -s -X POST "$BASE/run/end" \
  -H "X-Sign-Secret: $SECRET" -H "Content-Type: application/json" \
  -d '{"deviceId":"73F156FD","score":1234}'

# 6. 查 run 状态 (轮询)
curl -s -H "X-Sign-Secret: $SECRET" "$BASE/run/<runId>/status"
```

PowerShell 版本只需把 `\` 换成 `` ` `` 续行符,`'...'` 替换为 `"..."` 注意转义。
